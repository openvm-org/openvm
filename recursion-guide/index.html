The Recursion Sub-Circuit

The recursion sub-circuit verifies a SWIRL proof inside a SWIRL circuit. Instantiated with a child verifying key vk_child, its 39 AIRs constrain every phase of the reference verifier: preamble, GKR, batch constraint evaluation, stacking reduction, and WHIR. This guide lays out the full correspondence — AIR by AIR, bus by bus — and walks through the proof that any satisfying witness encodes a valid child proof.

Architecture

The circuit mirrors the reference verifier's sequential phases. Each module owns a contiguous range of transcript indices and hands off to the next via a module bus. The Transcript module and Primitive Lookup Tables are shared across every phase.

Proof Walkthrough

Each phase is a single page showing the Rust Air::eval implementations in source order, with prose explanations between code chunks. Hover a code chunk to highlight the cells and draw the bus connections it discharges; click to pin; click again (or elsewhere) to unpin.

AIR Browser

Every AIR in the recursion circuit. Each card visualizes the AIR as a trace table with labeled columns. Buses are listed beneath each AIR with their direction (S=send, R=receive, P=provide, L=lookup).

Bus Browser

Every bus in the circuit — permutation buses (multiset-equality between senders and receivers) and lookup buses (existence checks on tables). Each card shows the message format and both sides.
